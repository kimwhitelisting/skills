# frontend design
## 빠른 안내

- **문서 요약**: 이 기술은 일반적인 "AI 슬롭" 미학을 피하고, 의도와 완성도가 있는 프로덕션급 프런트엔드 인터페이스를 만들기 위한 가이드입니다.
- **핵심 섹션**: 한눈에 실행 순서, 디자인적 사고, 구현 목표, 프론트엔드 미학 지침, 피해야 할 패턴 (AI 슬롭), 품질 기준

### 권장 읽기 순서
1. 한눈에 실행 순서
2. 디자인적 사고
3. 구현 목표
4. 프론트엔드 미학 지침
5. 피해야 할 패턴 (AI 슬롭)



이 기술은 일반적인 "AI 슬롭" 미학을 피하고,
의도와 완성도가 있는 프로덕션급 프런트엔드 인터페이스를 만들기 위한 가이드입니다.

사용자는 컴포넌트, 페이지, 앱 전체 등 구현 대상을 요청합니다.
요청에는 목적, 대상 사용자, 기술 제약(프레임워크/성능/접근성)이 함께 포함될 수 있습니다.

## 한눈에 실행 순서

1. 맥락 정리: 목적과 사용자를 먼저 정의
2. 미학 방향 선택: 하나의 톤을 명확히 고정
3. 제약 확인: 기술 요구사항을 디자인에 반영
4. 차별화 정의: 기억에 남을 핵심 포인트 1개 결정
5. 구현 및 검증: 코드 품질과 시각 완성도를 함께 점검

## 디자인적 사고

### 1) 목적과 사용자 정의
- **목적**: 이 인터페이스는 어떤 문제를 해결하는가?
- **사용자**: 누가, 어떤 상황에서 사용하는가?

### 2) 톤 결정(극단을 선택)
- 극도로 미니멀 / 최대주의적 카오스
- 복고풍 미래지향 / 유기적·자연적
- 럭셔리·세련 / 장난기·토이 감성
- 사설·매거진 / 브루탈리즘·원시
- 아르데코·기하학 / 소프트·파스텔 / 산업·실용

영감을 참고하되, **선택한 방향에 끝까지 충실하게** 구현합니다.

### 3) 제약조건 확정
- 프레임워크
- 성능 목표
- 접근성 기준

### 4) 차별화 포인트 1개
- "왜 이 화면이 기억에 남는가?"에 대한 답을 하나로 명확히 정합니다.

!!! tip "핵심 원칙"
    대담한 맥시멀리즘도, 정제된 미니멀리즘도 유효합니다.
    중요한 것은 강도가 아니라 **의도**와 **일관성**입니다.

## 구현 목표
- 생산 등급의 기능성
- 시각적으로 눈에 띄고 기억에 남는 결과물
- 미학적 관점이 분명하고 응집력 있는 UI
- 디테일까지 정제된 마감

## 프론트엔드 미학 지침

### 타이포그래피
- Arial, Inter 같은 과사용 기본 조합은 피합니다.
- 독특한 디스플레이 폰트 + 안정적인 본문 폰트를 조합합니다.
- 글꼴 선택만으로도 방향성이 보이게 만듭니다.

### 색상/테마
- CSS 변수로 색상 체계를 고정합니다.
- 포인트가 분명한 팔레트가 흐릿한 팔레트보다 좋습니다.
- 톤은 많아도 되지만, 원칙은 단순해야 합니다.

### 모션
- 핵심 순간 중심으로 애니메이션을 설계합니다.
- HTML은 CSS 우선, React는 Motion 라이브러리 활용을 권장합니다.
- 분산된 미세 효과보다 구조적 페이지 리빌(지연 등장)이 더 효과적일 수 있습니다.

### 공간 구성
- 예상 가능한 레이아웃에서 벗어납니다.
- 대각선 흐름, 그리드 브레이크, 네거티브 스페이스를 의도적으로 활용합니다.
- 밀도는 "많음/적음"이 아니라 "통제됨/무질서"로 판단합니다.

### 배경/시각 디테일
- 단색 배경 대신 분위기와 깊이를 만듭니다.
- 그라디언트 메쉬, 노이즈 텍스처, 패턴, 투명 레이어, 그림자, 장식 테두리, 커스텀 커서, 그레인 오버레이 등을 상황에 맞게 사용합니다.

## 피해야 할 패턴 (AI 슬롭)
- 과사용 글꼴 조합(Inter, Roboto, Arial, 시스템 기본 글꼴)
- 진부한 색 구성(예: 흰 배경 + 보라 그라데이션)
- 예측 가능한 레이아웃/컴포넌트 반복
- 맥락성이 없는 템플릿형 화면

## 품질 기준
- 구현 복잡도는 미학 목표에 맞춰 조절합니다.
- 맥시멀리즘이면 애니메이션/효과까지 완성합니다.
- 미니멀리즘이면 간격·타이포·미세 디테일 정밀도가 핵심입니다.

기억하세요. 특별한 결과는 "새로운 스타일"보다
**명확한 비전의 일관된 실행**에서 나옵니다.
