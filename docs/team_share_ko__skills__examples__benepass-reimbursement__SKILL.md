## 문서 정보

- **이름**: `benepass-reimbursement`
- **설명**: Benepass(app.getbenepass.com)를 통해 비용 환급을 제출하세요. 고용주가 Benepass를 혜택 플랫폼으로 사용하는 사용자를 위한 것입니다. 로그인,
혜택 선택, 양식 작성, 영수증 업로드 및 제출을 처리합니다. 브라우저/컴퓨터 사용 기능이 필요합니다.
- **라이선스**: -

# 베네패스 환급 스킬

브라우저 자동화, 인증 코드용 Gmail 통합, 영수증용 파일 업로드를 사용하여 로그인부터 제출까지 전체 Benepass 환급 흐름을 자동화하세요.

---

## 전제 조건

- Benepass 탐색을 위해 **브라우저 액세스**(컴퓨터 도구)를 활성화합니다.
- **코드 실행 및 파일 생성** 활성화 - 영수증 파일 저장 및 업로드에 필요
- 이메일 확인 코드를 가져오기 위해 **Gmail MCP** 구성
- Gmail MCP를 통해 이메일을 결정합니다. Gmail에 액세스할 수 없으면 요청하세요.
- 영수증 이미지 얻기(스크린샷, 사진, PDF)

---

## 1단계: 영수증 세부정보 추출

Benepass로 이동하기 전에 영수증 이미지나 메시지에서 주요 세부정보를 추출하세요.

- **금액**(예: $84.99)
- **판매자**(예: Verizon, DoorDash, Lyft)
- **메모/메모** — 제공된 메모를 적용하거나 기본적으로 간단한 설명(예: "집 Wi-Fi", "점심", "회사로 이동")을 기본으로 설정합니다.
- **혜택 카테고리** — 지정된 카테고리를 선택합니다. 확신이 있으면 맥락에서 추론하고, 그렇지 않으면 설명을 요청합니다(4단계 참조).

---

## 2단계: Benepass에 로그인하세요.

### 2a. 이메일 탐색 및 입력

```
Navigate to: https://app.getbenepass.com
```

- 이메일 입력란을 클릭하고 이메일 주소를 입력하세요.
- 두 가지 로그인 옵션이 나타납니다: **"G-Suite로 로그인"** 및 **"이메일 코드로 로그인"**
- **"이메일 코드로 로그인"**을 클릭하세요.

### 2b. Gmail에서 인증 코드 가져오기

6개의 입력 상자가 나타나는 "인증 코드 입력" 프롬프트를 기다립니다.

Gmail MCP를 통해 코드를 가져옵니다.

```
Gmail search query: "from:benepass verification code"
maxResults: 1
```

- 이메일에서 6자리 코드를 추출합니다('인증 코드는 XXXXXX' 검색).
- 인증 입력란에 6자리 코드를 입력하세요. 6자리 숫자를 모두 입력하면 페이지가 자동으로 제출됩니다.
- 대시보드가 ​​로드될 때까지 3~5초 정도 기다립니다.

### 2c. 로그인 성공 확인

로그인 성공을 확인하기 위해 인사말, 계정 잔액, 인사이트 카드로 대시보드가 ​​로드되는지 확인하세요.

**중요**: 만료된 인증 코드를 다음과 같이 처리하세요.

- 인증페이지에서 '받지 못하셨나요? 재전송'을 클릭하세요.
- 5초 정도 기다린 후 Gmail에서 다시 새로운 코드를 검색해 보세요.

---

## 3단계: 환급 시작

- 왼쪽 사이드바에서 **"환급 받기"** 버튼을 클릭하세요.
- 환급 양식이 다음 두 섹션으로 로드될 때까지 기다립니다.
1. **혜택 선택**(드롭다운)
2. **상세정보 입력** (금액, 가맹점, 메모, 영수증)

---

## 4단계: 혜택 선택

- **"혜택 선택"** 드롭다운을 클릭하세요.
- 지정된 카테고리를 선택하세요. 확신이 있으면 상황에서 추론하고, 그렇지 않으면 설명을 요청합니다.
- 드롭다운에서 직접 사용 가능한 혜택 옵션을 읽어보세요. 카테고리는 고용주에 따라 다릅니다.
- 선택한 혜택에 표시된 잔액이 환급금액에 해당하는지 확인하세요.

---

## 5단계: 세부정보 입력

### 금액 필드

**중요**: 올바른 금액을 입력하기 전에 금액 필드에서 미리 입력된 값을 모두 지웁니다.

- 금액 필드를 세 번 클릭하여 기존 값을 선택합니다.
- 정확한 금액을 입력하세요(예: `84.99`).
- `$` 기호를 포함하지 말고 숫자만 입력하세요.

### 판매자 필드

- 가맹점 입력란을 클릭하세요.
- 판매자 이름을 입력하세요(예: `Verizon`, `DoorDash`, `Lyft`).

### 메모 필드(선택사항이지만 권장됨)

- 메모 입력란을 클릭하세요.
- 메모를 입력하세요. (예: `Home wifi`, `Lunch`, `Ride to office`)

---

## 6단계: 영수증 업로드

**중요**: 파일 입력이 `id="fileInput"`이 포함된 **숨겨진 요소**입니다. 보이는 드롭 영역은 클릭할 수 없습니다. 대신 숨겨진 입력을 사용하세요. `id="fileInput"`이 포함된 요소가 없으면 페이지에서 `<input type="file">`을 검색하세요.

1. `read_page`을 `filter: interactive`과 함께 사용하여 파일 입력 요소를 찾습니다.
2. 검색: `<input id="fileInput" class="hidden" type="file">`
3. 숨겨진 입력에 대한 참조와 함께 `upload_file` 도구를 사용합니다.

```
upload_file:
  file_path: <path to the receipt file>
  ref: <ref for fileInput>
```

4. 업로드 후 다음을 확인합니다.
- **영수증** 라벨이 더 이상 빨간색이 아닙니다(업로드 전에는 빨간색이었습니다).
- 파일 이름과 크기를 보여주는 "업로드된 파일" 섹션이 나타납니다.
- 영수증의 썸네일 미리보기가 표시됩니다.

---

## 7단계: 제출

- 아래로 스크롤하여 **"환급 제출"** 버튼을 확인하세요.
- **제출을 클릭하기 전에** 확인을 위해 요약을 제시하세요.
- 혜택 항목
- 양
- 상인
- 메모/메모
- 영수증 파일명
- 제출 후 남은 혜택 잔액
- 계속하기 전에 명시적인 승인을 기다리십시오.
- **"환급 제출"**을 클릭하세요.
- 처리를 위해 3초간 기다립니다.

### 제출 성공 확인

제출 후 페이지는 다음을 보여주는 비용 세부 정보 보기로 리디렉션됩니다.

- **상태**: 보류 중
- **결제수단**: 환급
- **혜택**: 선택한 혜택명
- **지급 진행 상황**: 미결제 금액 = 제출 금액

확인을 보고합니다.

## 문제 해결

### 세션이 만료되었습니다

오류가 발생하거나 페이지가 로그인으로 리디렉션되면 **2단계**(로그인)부터 다시 시작하여 세션 만료를 처리하세요.

### 인증코드가 작동하지 않습니다

만료된 인증 코드 처리: '받지 못하셨나요? 재전송'을 클릭하고 몇 초간 기다린 후 Gmail에서 최신 코드를 다시 검색하세요. **가장 최근** 이메일을 가져왔는지 확인하세요(기본 정렬로 `maxResults: 1` 사용).

### 금액 필드 문제

새 금액을 입력하기 전에 항상 금액 필드를 세 번 클릭하여 기존 텍스트를 모두 선택하세요. 이렇게 하면 연결(예: `32184.99`)이 방지됩니다.

### 업로드가 등록되지 않았습니다

영수증 업로드가 작동하지 않는 경우:

1. `read_page`이 포함된 페이지를 다시 읽어 `fileInput` 참조를 찾습니다.
2. `fileInput`을 찾을 수 없으면 `<input type="file">` 요소를 검색합니다.
3. 올바른 참조를 사용하여 `upload_file`으로 다시 업로드해 보세요.
4. 업로드 후 "업로드된 파일" 섹션이 나타나는지 확인하세요.

### 혜택 잔액이 너무 낮습니다.

선택한 혜택의 잔액이 부족한 경우 문제를 보고하고 다른 혜택 카테고리를 사용할지 문의하세요.
